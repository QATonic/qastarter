# Getting Help with QAStarter

We want to make it easy for you to get help — whether you're stuck on a
template, hit a generation bug, or have an idea you want to bounce off
the maintainers.

## Pick the right channel

| Need | Where to go |
|---|---|
| **Ask a question** ("how do I…", "is X supported?") | [GitHub Discussions](https://github.com/QATonic/QAStarter/discussions) |
| **Report a bug** (something is broken or wrong) | [Open an issue](https://github.com/QATonic/QAStarter/issues/new/choose) using the Bug Report template |
| **Request a feature or new template** | [Open an issue](https://github.com/QATonic/QAStarter/issues/new/choose) using the Feature Request or Template Request template |
| **Found a security vulnerability?** | Read [SECURITY.md](SECURITY.md) — please do **not** open a public issue |
| **Want to contribute code?** | Start with [CONTRIBUTING.md](CONTRIBUTING.md) |
| **Documentation question** | Start at [the Docs site](https://qastarter.qatonic.com/docs) |
| **General chat / community** | LinkedIn / X / YouTube — links in the website footer |

## Before you file

A quick checklist so we can help you faster:

- **Search first.** Someone may have already asked. Check the
  [issue tracker](https://github.com/QATonic/QAStarter/issues?q=is%3Aissue) and
  [Discussions](https://github.com/QATonic/QAStarter/discussions).
- **Reproduce locally if you can.** Generate a fresh project from the
  live site or `npm run dev` and confirm the issue isn't a stale local
  copy.
- **Use the templates.** The Bug / Feature / Template-request issue
  forms ask for the testing type, framework, language, and runner — that
  combo uniquely identifies the pack we need to look at.
- **Include version.** `package.json` version + the deployed site you
  used + the exact `/api/v1/generate?…` query string if you hit the API.

## Response expectations

QAStarter is maintained by [QATonic Innovations](https://qatonic.com)
and the OSS community. We aim to triage new issues within 5 working
days. Maintainer time is finite — sponsorship via
[GitHub Sponsors](https://github.com/sponsors/QATonic) or
[Buy Me a Coffee](https://www.buymeacoffee.com/santoshkarad) helps
keep the project moving.

If your need is commercial / SLA-bound, see the website footer for
QATonic's contact details.

## Maintainers

The active code-owners are listed in
[`.github/CODEOWNERS`](.github/CODEOWNERS) (when present) — they
auto-receive review requests on PRs. For everything else, the
fastest route is GitHub Discussions or Issues.
