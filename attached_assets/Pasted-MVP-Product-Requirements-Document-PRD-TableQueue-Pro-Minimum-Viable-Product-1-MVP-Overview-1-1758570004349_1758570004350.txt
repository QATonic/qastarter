MVP Product Requirements Document (PRD): TableQueue Pro - Minimum Viable Product
1. MVP Overview
1.1 Product Name
TableQueue Pro MVP - Digital Queue Management System

1.2 MVP Focus
Core functionality that solves the most critical pain points with minimal complexity and maximum reliability.

1.3 MVP Philosophy
"Do one thing well" - Focus exclusively on replacing the notebook-based queue management system with a digital solution that provides immediate value.

2. MVP Scope & Boundaries
2.1 IN Scope for MVP
Digital customer registration and queue management

Real-time table status tracking

Basic SMS notifications to customers

Simple queue display for staff

Single-restaurant setup

2.2 OUT of Scope for MVP
Advanced analytics and reporting

Customer mobile app

Reservation system integration

Multi-language support

Multiple location management

WhatsApp integration

Loyalty programs

Digital menus

3. MVP User Stories & Requirements
3.1 Core User Flow
As a Gatekeeper, I want to:

Quickly add customers to the digital queue with their name, phone number, and group size

See the current queue status at a glance

Update customer status when their table is ready

View available tables that match customer group size

As a Waiter, I want to:

Mark tables as occupied/available with one tap

See which tables I'm responsible for

Get notified when new customers are assigned to my tables

As a Customer, I want to:

Receive an SMS with my queue position and estimated wait time

Get notified when my table is ready

Check my queue status without bothering staff

As a Restaurant Manager, I want to:

Set up my restaurant layout quickly

View basic queue metrics at the end of the day

Manage staff access to the system

5.2 Database Schema (MVP Simplified)
sql
-- Restaurants
CREATE TABLE restaurants (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    phone VARCHAR(15),
    created_at TIMESTAMP DEFAULT NOW()
);

-- Tables
CREATE TABLE tables (
    id SERIAL PRIMARY KEY,
    restaurant_id INTEGER REFERENCES restaurants(id),
    table_number VARCHAR(10) NOT NULL,
    capacity INTEGER NOT NULL,
    status VARCHAR(20) DEFAULT 'available' -- available, occupied, cleaning
);

-- Queue
CREATE TABLE queue (
    id SERIAL PRIMARY KEY,
    restaurant_id INTEGER REFERENCES restaurants(id),
    customer_name VARCHAR(100) NOT NULL,
    phone_number VARCHAR(15) NOT NULL,
    group_size INTEGER NOT NULL,
    queue_position INTEGER NOT NULL,
    status VARCHAR(20) DEFAULT 'waiting', -- waiting, ready, seated, cancelled
    estimated_wait INTEGER,
    created_at TIMESTAMP DEFAULT NOW(),
    seated_at TIMESTAMP
);

-- Staff
CREATE TABLE staff (
    id SERIAL PRIMARY KEY,
    restaurant_id INTEGER REFERENCES restaurants(id),
    name VARCHAR(100) NOT NULL,
    role VARCHAR(20) NOT NULL, -- admin, gatekeeper, waiter
    pin_code VARCHAR(4) NOT NULL -- for quick login
);
5.3 API Endpoints (MVP Core)
Queue Management

POST /api/queue/add - Add customer to queue

GET /api/queue/current - Get current queue

PUT /api/queue/:id/status - Update customer status

DELETE /api/queue/:id - Remove from queue

Table Management

GET /api/tables - Get all tables with status

PUT /api/tables/:id/status - Update table status

POST /api/tables/assign - Assign customer to table

SMS Management

POST /api/sms/send - Send SMS notification

GET /api/sms/templates - Get SMS templates

9. MVP Limitations & Workarounds
9.1 Known Limitations
No offline mode (requires stable internet)

Basic SMS only (no WhatsApp)

Single device support initially

No advanced reporting

9.2 Workarounds
Mobile hotspot as backup internet

Manual customer calling if SMS fails

Scheduled exports for basic reporting

Multiple staff can use same device with different PINs
