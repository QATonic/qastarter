# WhatsApp Launch Messages

> **Hard rules for WhatsApp.** Unlike LinkedIn or Twitter, WhatsApp is
> someone else's group. You're a guest.
> - Never post in a group you haven't been active in for 30+ days
> - Always lead with the "help"; never lead with the link
> - One message, not a thread. No follow-ups unless someone replies
> - Don't cross-post the exact same text in 5 groups the same hour — it
>   looks (and flags) as spam
> - Link previews only render if the link is first or alone on a line
>
> **Where to post (in order of priority):**
> 1. QA Slack/Discord groups you're already active in (not WhatsApp, but start there)
> 2. "QA Engineers" / "Selenium Community" / "Playwright Users" WhatsApp groups you've contributed to
> 3. Your company's internal test-automation channel (if the team would genuinely benefit)
> 4. Regional QA groups (India, Brazil, Philippines all have large testing communities on WhatsApp)

## After-post playbook

- Let the link preview render before you type anything else
- If someone replies with a question, answer them with substance. Don't push the link again.
- If someone says "cool, will try" — reply with a specific onboarding tip
  ("tip: start with `qastarter list --type web` to see the 22 web combos")
- If 3 people ignore it — don't repost to the same group for 30 days

## What NOT to send on WhatsApp

- ❌ Don't send a long thread across 5 messages — each one bumps the group
- ❌ Don't send "Please give feedback 🙏🙏🙏" as a follow-up
- ❌ Don't share the marketing/launch/whatever.pdf as an attachment
- ❌ Don't tag @admins asking for pinned posts unless you're actually an admin
